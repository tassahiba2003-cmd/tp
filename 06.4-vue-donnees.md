# §6.4 — Vue des données

## Modèle entité-relation

Le modèle de données de **SupEvents** est conçu pour structurer les échanges entre les étudiants, les organisateurs et les services de paiement tiers [cite : 1].  
Il privilégie une approche sémantique où chaque entité représente un concept métier clair, facilitant ainsi l’interopérabilité des services [cite : 1].  
Cette modélisation assure la traçabilité complète du cycle de vie d’un ticket, de sa réservation à sa confirmation finale après paiement.

```mermaid
erDiagram
    USER ||--o{ TICKET : "achète"
    USER ||--o{ NOTIFICATION : "reçoit"
    ORGANIZER ||--o{ EVENT : "organise"
    EVENT ||--o{ TICKET : "propose"
    TICKET ||--o| PAYMENT : "génère"
    CATEGORY ||--o{ EVENT : "catégorise"

    USER {
        uuid id PK
        string sso_subject_id UK
        string email UK
        string role
        timestamp created_at
    }
    ORGANIZER {
        uuid id PK
        uuid user_id FK
        string company_name
        string status
    }
    CATEGORY {
        uuid id PK
        string name UK
    }
    EVENT {
        uuid id PK
        uuid organizer_id FK
        uuid category_id FK
        string title
        timestamp start_date
        string visual_url
        string status
    }
    TICKET {
        uuid id PK
        uuid user_id FK
        uuid event_id FK
        string status
        decimal price
        timestamp purchased_at
    }
    PAYMENT {
        uuid id PK
        uuid ticket_id FK
        string payment_intent_id UK
        string status
        decimal amount
    }
    NOTIFICATION {
        uuid id PK
        uuid user_id FK
        string type
        string message
        timestamp sent_at
    }
