# §6.4 — Vue des données

## 1. Modèle entité-relation
Ce bloc de conception technique pour **SupEvents** définit la structure sémantique des données persistées par le système.  
Contrairement à un simple schéma physique, cette section explicite les décisions architecturales qui garantissent l’interopérabilité des services [cite: 1].  
Nous documentons ici le **modèle logique** qui supporte les flux transactionnels critiques (inscriptions, paiements).  
L’objectif est de rendre visibles les invariants métier afin de faciliter les développements futurs et la maintenance.

erDiagram

    TICKET ||--o| PAYMENT : "génère"
